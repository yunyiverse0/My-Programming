# 🛒 통합 장바구니 웹앱 (Personal Shopping Hub)

## 1. 서비스 개요

본 서비스는 **결제를 직접 처리하지 않는 개인용 통합 장바구니 웹앱**이다.  
사용자는 여러 브랜드 및 쇼핑 플랫폼(무신사, 지그재그, 에이블리 등)에 흩어져 있는 상품을 한 곳에 모아 관리할 수 있으며,  
상품 선택 시 실제 결제는 각 플랫폼의 상품 페이지로 이동하여 진행한다.

즉, 이 서비스는 쇼핑몰이 아닌 **“구매 전 정리·비교·관리”를 위한 개인 쇼핑 허브**를 목표로 한다.

---

## 2. 서비스 정체성 (Positioning)

- ❌ 자체 결제 / 주문 / 재고 관리 없음
- ❌ 입점 계약 및 제휴 불필요
- ⭕ URL 기반 상품 관리
- ⭕ 플랫폼 중립적 구조
- ⭕ 개인 사용자 중심의 장바구니 / 위시리스트

### 한 줄 정의
> 여러 쇼핑 플랫폼의 상품을 한 곳에 모아보고,  
> 최종 구매는 사용자가 선택한 플랫폼에서 진행하는 개인 쇼핑 관리 웹앱

---

## 3. 핵심 기능 요약

### 3.1 상품 저장 (장바구니)
- 상품명, 브랜드명, 카테고리(상의/하의 등)
- 대표 이미지 URL
- 플랫폼별 상품 URL 저장
  - 무신사
  - 지그재그
  - 에이블리 등

> 실제 상품 데이터가 아닌 **상품 메타 정보 + 링크**만 관리

---

### 3.2 플랫폼 선택 후 이동
- 상품이 **한 플랫폼에만 존재**할 경우  
  → 해당 플랫폼 상품 페이지로 바로 이동

- 상품이 **여러 플랫폼에 존재**할 경우  
  → 플랫폼 선택 화면 제공  
  → 선택한 플랫폼의 상품 URL로 이동

---

### 3.3 카테고리(섹션) 분리
- 사용자가 상품을 카테고리별로 분류 가능
  - 상의
  - 하의
  - 아우터
  - 신발
  - 가방 등
- 카테고리별 장바구니 필터링 제공

---

### 3.4 상품 검색 기능
- 사용자가 장바구니에 담은 상품을 대상으로 검색
- 검색 기준:
  - 상품명
  - 브랜드명
  - 카테고리

> 외부 쇼핑몰 자동 검색이나 크롤링은 포함하지 않음

---

## 4. 사용자 흐름 (User Flow)

1. 사용자가 상품 정보를 직접 입력하거나 URL 기반으로 추가
2. 장바구니에서 상품을 카테고리별로 관리
3. 상품 클릭
   - 플랫폼 1개 → 즉시 이동
   - 플랫폼 2개 이상 → 플랫폼 선택 화면 표시
4. 선택한 플랫폼의 상품 페이지로 이동 후 결제 진행

## 5. 기술적 설계 방향

### 5.1 프론트엔드
- HTML / CSS / JavaScript
- 카드 기반 상품 UI
- 필터링 및 검색 기능

### 5.2 백엔드
- Flask (Python)
- 기능:
  - 장바구니 데이터 관리
  - 상품 추가 / 조회
  - 플랫폼 선택 처리

### 5.3 데이터 관리
- 초기: 메모리 기반 임시 데이터
- 확장:
  - MySQL 연동
  - 사용자별 장바구니 관리

## 6. 제외 범위 (명확한 한계)

- 결제 기능
- 주문 / 배송 / CS 처리
- 실시간 가격 비교
- 자동 상품 수집(크롤링)
- 플랫폼 API 연동

---

## 7. 기대 효과 및 활용성

- 여러 쇼핑 플랫폼을 동시에 사용하는 사용자에게 높은 편의성 제공
- 학생 프로젝트 및 포트폴리오로 활용 가능
- 향후 확장 가능:
  - 가격 메모
  - 구매 여부 체크
  - 찜/보류 상태 관리
  - 개인 스타일 기반 카테고리

---

## 8. 프로젝트 성격

- 개인 프로젝트
- 학습 목적의 웹 애플리케이션
- 실제 결제/상업 서비스 목적 아님
